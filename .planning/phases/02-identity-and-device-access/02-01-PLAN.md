  <behavior>
    - Missing or malformed Authorization header returns 401 with code AUTH_TOKEN_MISSING_OR_MALFORMED and generic message Authentication failed.
    - Expired token returns 401 with code AUTH_TOKEN_EXPIRED and generic message Authentication failed.
    - Invalid issuer, audience/client_id, token_use, or signature returns 401 with code AUTH_TOKEN_INVALID_CLAIMS.
    - Only valid token with insufficient policy returns 403 with code AUTH_FORBIDDEN.
    - Valid token payload is returned for downstream handlers.
  </behavior>
  <action>Implement CognitoJwtVerifier singleton using env-driven userPoolId and appClientId. Implement strict bearer extraction in auth guard and map verifier outcomes through one auth-error-mapper module that enforces locked contract: 401 for missing/invalid, 403 only for authorized-token-but-forbidden cases, explicit expired-token code, and generic human-facing message. Encode machine codes exactly: AUTH_TOKEN_MISSING_OR_MALFORMED, AUTH_TOKEN_EXPIRED, AUTH_TOKEN_INVALID_CLAIMS, AUTH_FORBIDDEN. Keep mapping reusable by all protected routes.</action>
